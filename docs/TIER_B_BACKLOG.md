### Phase F acceptance (draft — tighten as C–E advance)

Claiming **full** `azl_interpreter.azl` **behavior** on the semantic spine requires **all** of the following (process + artifacts; not init-only smoke):

1. **Trace:** `AZL_RUNTIME_SPINE=azl_interpreter` (or `semantic`) runs **`tools/azl_runtime_spine_host.py`** → **`minimal_runtime.py`**; **Gate G2** still passes (**`AZL_SPINE_EXEC_OWNER=minimal_runtime_python`** = **current** transitional **exec carrier on that launcher only**; **`AZL_SEMANTIC_SPEC_OWNER`** = **`azl_interpreter.azl`** = **target** semantic spec — **not** “Python owns language meaning,” **not** default **C minimal** retired, **not** a fake transition to **C orchestrates; AZL interprets** as the sole lived path until product/spine policy says so).
2. **Interpreter file:** Combined program (stub security + entry + real **`azl/runtime/interpreter/azl_interpreter.azl`**) **`behavior`** listeners execute without **silent** failure; failures use **ERROR_SYSTEM**-shaped behavior where applicable.
3. **Semantics:** **`::tokenize_line`** / **`::parse_tokens`** / execution path match the **in-file** interpreter for the **claimed** statement subset — proven by **F-gates** on fixtures **and** at least one **extended smoke** or **`verify_*`** that exercises **more lines** of the real file than steps **3–4** alone.
4. **Parity:** **`scripts/check_azl_native_gates.sh`** green through the current **F** range on **`p0_semantic_*.azl`**; new exits documented in **ERROR_SYSTEM.md**.

**Benchmarks:** **`make benchmark-real-world`** and **`make benchmark-azl-full-report`** are **rulers** and **integration timing**; they do **not** prove AZL language speed on Benchmarks Game workloads — see [BENCHMARKS_AZL_VS_REAL_WORLD.md](BENCHMARKS_AZL_VS_REAL_WORLD.md).
